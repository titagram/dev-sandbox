# FEATURE.md

Use when adding new behavior to the target project.

## Required Reads

- `ai-sandbox/config/project.yaml`
- `ai-sandbox/wiki/README.md`
- `ai-sandbox/instructions/policies/FILE_BOUNDARIES.md`
- `ai-sandbox/instructions/policies/SOURCE_OF_TRUTH.md`

## Process

1. Clarify user goal and acceptance criteria.
2. Check existing architecture and patterns.
3. Identify write paths and protected paths.
4. Add tests proportional to risk.
5. Implement in the project style.
6. Update wiki and logbooks.

## Completion

- Feature behavior is verified.
- Documentation reflects the new behavior.
- Residual risks are recorded.
