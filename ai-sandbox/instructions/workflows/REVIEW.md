# REVIEW.md

Use when reviewing project code, sandbox changes, or generated documentation.

## Required Reads

- `ai-sandbox/config/project.yaml`
- relevant workflow docs for the reviewed area

## Review Order

1. Findings first, ordered by severity.
2. File and line references when available.
3. Missing tests and verification gaps.
4. Open questions.
5. Short summary only after findings.

## Completion

- No code is changed unless the developer explicitly asks for fixes.
- Findings distinguish verified issues from risks and assumptions.
