# EXISTING_PROJECT.md

Use these questions after the developer says the target is an existing project.

1. Which modules are known to be fragile?
2. Which bugs or regressions happened recently?
3. Which documentation is reliable?
4. Which documentation is stale or misleading?
5. Which test commands are fast and trustworthy?
6. Which test commands are slow, flaky, or unsafe?
7. Which runtime services are required locally?
8. Which database, queue, cache, or storage services are required?
9. Which deploy process is used?
10. Which code style or legacy conventions must be preserved?
11. Which paths contain generated files and must not be edited manually?
12. Which areas require extra approval before modification?

For existing projects, do not rewrite architecture or modernize style unless the developer explicitly asks for that work.
