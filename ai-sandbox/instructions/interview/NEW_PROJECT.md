# NEW_PROJECT.md

Use these questions after the developer says the target is a new project.

1. What product or internal tool are we building?
2. Who are the primary users?
3. What is the smallest useful MVP?
4. Which stack is preferred?
5. Which stack is forbidden?
6. Where will it be deployed?
7. What data must be persisted?
8. Is authentication required?
9. Which external integrations are expected?
10. What test coverage is required before the project is considered usable?
11. What documentation should be generated by default?
12. Which non-functional requirements matter: performance, privacy, auditability, accessibility, offline use, or localization?

Record assumptions as `developer_provided` only when explicitly confirmed by the developer.
